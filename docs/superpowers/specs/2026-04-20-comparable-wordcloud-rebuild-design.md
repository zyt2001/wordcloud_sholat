# 双侧可比词云重构设计

## 1. 重构原因

旧版实现存在三个根本问题：

1. 数据口径不对齐。`模拟网站用户` 原始数据覆盖 `1990-2025`，但旧页面只消费了 3 个历史 `txt`，导致大量时间段看似缺数据，实际上只是未被重新整理。
2. 视觉风格过重，不符合当前答辩页需要的简洁展示感。
3. 词云布局直接把大量细粒度标签放进有限画布，造成明显重叠。

因此本轮不再在旧页面上继续修补，而是用 `src/` 下的新实现重建一版真正可比、风格更简洁的答辩词云页面。

## 2. 新版目标

1. 从原始数据重建 `2005-2024` 的双侧可比主题数据。
2. 统一展示名称为：`模拟学院用户`、`模拟网站用户`。
3. 新代码全部放入新建的 `src/` 目录，与旧文件隔离。
4. 页面改为简洁、留白更多、答辩友好的风格。
5. 通过更粗的公共主题标签和更大的展示空间，消除词云重叠。

## 3. 数据口径

### 3.1 数据源

新版正式数据源为：

1. `社交网络应用/研究兴趣动态词云/datas/*_papers_by_year.json`，对应 `模拟学院用户`
2. `社交网络应用/研究兴趣动态词云/arxiv_year3/*_papers.json`，对应 `模拟网站用户`

历史 `txt` 文件仅作为参考，不再作为新版前端的主输入。

### 3.2 时间范围

主研究范围固定为：`2005-2024`

原因：

1. `模拟学院用户` 原始数据从 2005 年开始才形成有效覆盖。
2. `模拟网站用户` 在该区间内也拥有完整的按年原始数据。
3. 这样可以保证时间轴中的所有分段都拥有双侧真实样本，不再人为制造 `partial/pending` 主体页面。

### 3.3 时间分段

新版时间分段改为更适合学院侧样本量的 6 段：

1. `2005-2011`
2. `2012-2016`
3. `2017-2019`
4. `2020-2021`
5. `2022-2023`
6. `2024`

这样既保留演化过程，又避免学院侧早期按单年切分时样本过少。

### 3.4 公共主题标签

新版不直接展示大量细碎方向词，而是将论文归并到一套更粗的公共主题标签：

1. `人工智能与机器学习`
2. `计算机视觉与多媒体`
3. `自然语言与语音`
4. `知识计算与数据智能`
5. `系统、软件与体系结构`
6. `网络、通信与安全`
7. `算法理论与形式化`
8. `交叉应用与智能场景`

映射依据由两部分构成：

1. arXiv `categories` 字段
2. 标题与摘要中的关键词规则

这样做的目的不是追求最细粒度分类，而是确保双侧口径统一、对比真实、页面不拥挤。

## 4. 数据产物

新版输出一份前端消费数据：

`src/web/data/comparison_timeline.json`

每个 segment 至少包含：

1. `label`
2. `academy`
3. `website`
4. `summary`

其中每侧包含：

1. `name`
2. `sampleCount`
3. `topics`

每个 topic 至少包含：

1. `text`
2. `value`
3. `share`
4. `state`
5. `x`
6. `y`

## 5. 新目录结构

新版代码全部进入 `src/`：

```text
src/
```

测试文件放在：

`tests/test_build_comparison_data.py`

旧文件保留作参考，但不再作为新版交付入口。

## 6. 页面形态

新版页面走“简洁答辩页”风格，不再使用深色控制台视觉。

页面结构：

1. 顶部：标题、说明、当前时间段
2. 中部：左右双侧大词云区域
3. 中间下方：简短结论卡片
4. 底部：时间段切换条

整体特征：

1. 浅色背景
2. 更少装饰
3. 更大的词云画布
4. 更强的文字可读性

## 7. 词云布局策略

重叠问题的主要根因是“词太多 + 标签太碎 + 画布太小”。新版同时做三件事：

1. 把展示词收敛为 8 个公共主题标签
2. 扩大双侧词云展示区
3. 采用固定槽位布局，而不是让文本自由堆叠

固定槽位布局含义是：

1. 为 8 个主题预定义一组不重叠的位置槽
2. 每个时间段按权重把主题填入这些槽
3. 相同主题跨时间段复用相同基础位置

这样既保留演化感，也不会再出现大量文字重叠。

## 8. 验收标准

新版完成后需要满足：

1. 页面主入口位于 `src/web/index.html`
2. 新代码文件位于 `src/` 内
3. `2005-2024` 所有展示分段均有双侧真实数据
4. 双侧名称使用 `模拟学院用户` 与 `模拟网站用户`
5. 页面风格较旧版明显更简洁
6. 词云无明显重叠
7. 本地静态服务和本机浏览器验证通过
